# 01. Scope Statement — Analysis and Design Phase

## 1. 현재 과제 범위

현재 과제는 사용자의 요구사항을 바탕으로 MSG Foundation 파일럿 소프트웨어 시스템을 **분석하고 설계하는 것**까지로 제한한다.

## 2. 수행할 작업

분석/설계 단계에서 수행할 작업은 다음과 같다.

1. PDF 요구사항 추출 및 원문 기반 요약
2. 문제 정의와 목표 상태 정리
3. 이해관계자 및 사용자 역할 분석
4. 기능/비기능 요구사항 도출
5. 도메인 용어와 비즈니스 규칙 정리
6. 데이터 요구사항과 데이터 모델 설계
7. 주간 자금 계산 로직 명세
8. 보고서 요구사항 명세
9. 시스템 경계와 아키텍처 설계
10. API 또는 내부 인터페이스 설계
11. UI/화면 흐름 초안 작성
12. 보안/권한/운영 고려사항 정의
13. 테스트 전략과 요구사항 추적표 작성
14. 구현 로드맵과 WBS 작성

## 3. 수행하지 않을 작업

현재 과제에서는 다음을 수행하지 않는다.

- 애플리케이션 코드 구현
- 데이터베이스 생성 또는 마이그레이션 실행
- 외부 API 연동 구현
- 서버 배포
- 실서비스 데이터 입력
- 실제 금융 의사결정 자동화
- PDF 요구사항에 없는 정책을 임의로 확정

## 4. 시스템 파일럿 범위

파일럿 시스템은 요구사항에 명시된 세 가지 데이터 범위만 다룬다.

1. Investment data
2. Operating expense data
3. Mortgage data

파일럿 시스템은 다음 세 가지 보고서를 지원해야 한다.

1. Weekly funds computation results
2. Listing of all investments
3. Listing of all mortgages

## 5. 주요 산출물

- 문제 정의서
- 범위 선언서
- 핵심 질문 목록
- 요구사항 명세서 / SRS
- 요구사항 추적표
- 도메인 모델
- 데이터 모델
- 계산 규칙 명세
- 보고서 명세
- 아키텍처 설계서
- 테스트 전략서
- 구현 로드맵 / WBS

## 6. 제약사항

- 수탁자들은 컴퓨터 경험이 없다고 가정한다.
- 파일럿 프로젝트 비용을 낮게 유지해야 한다.
- 주간 자금 계산에 필요한 데이터만 포함해야 한다.
- 일부 산식과 정책은 원문에서 모호하므로 검증 전 임의 확정하지 않는다.

## 7. 완료 조건

이 단계는 다음 조건을 만족하면 완료된다.

- 요구사항 원문에서 식별 가능한 모든 핵심 규칙이 문서화된다.
- 각 요구사항이 추적 가능한 ID를 가진다.
- 데이터, 계산, 보고서, 사용자 역할이 상호 모순 없이 연결된다.
- 구현자가 다음 단계에서 작업을 시작할 수 있는 수준의 설계 문서가 준비된다.
