# 01. Core Questions and Open Issues

이 문서는 분석/설계 진행 중 사용자, 교수자, 또는 도메인 담당자에게 확인해야 할 질문을 기록한다.

## 1. 주간 가용 자금 산식

### Q-001. 사용 가능한 금액 산식의 정확한 부호는 무엇인가?

원문은 다음 항목을 사용한다고 설명한다.

1. 주간 예상 투자 수익
2. 주간 예상 운영 비용
3. 해당 주의 총 예상 모기지 상환액
4. 해당 주의 총 예상 교부금

그러나 “주의 시작에 사용 가능한 금액은 위 항목 1, 2, 3, 4 이다”라고만 되어 있어 정확한 산식이 명확하지 않다.

가정 후보:

- `available = weekly investment income - weekly operating expenses + expected mortgage repayments - expected grants`

상태: `Needs Verification`

## 2. 100% 모기지 승인 조건

### Q-002. 90% 모기지 조건은 사전 자격 판단에만 쓰이는가?

MSG Foundation은 100% 모기지를 부여하지만, 조건 중 “90% 모기지에 대한 할부금 합산이 총소득의 28%를 초과하는 경우”가 포함되어 있다.

확인 필요:

- 90% 모기지는 일반 금융기관 이용 가능성을 판단하기 위한 비교 기준인가?
- 파일럿 시스템이 90% 모기지 P&I를 계산해야 하는가, 아니면 입력값으로 받는가?

상태: `Needs Verification`

## 3. 신규 신청 심사 범위

### Q-003. 파일럿 시스템은 신청 승인/거절까지 판단해야 하는가?

요구사항은 “주간 자금 계산에 필요한 데이터 항목만” 포함하라고 한다. 하지만 MSG Foundation의 모기지 부여 조건에는 결혼 기간, 고용 증거, 지역 주택 중간 가격, 저축액 등이 포함된다.

확인 필요:

- 파일럿 시스템은 이 자격 조건을 데이터로 관리해야 하는가?
- 아니면 자격 심사는 외부에서 수행되고, 시스템은 자금 가용성 계산만 담당하는가?

상태: `Needs Verification`

## 4. 보고서 출력 형태

### Q-004. 보고서는 화면 표시, 파일 출력, 인쇄 중 무엇을 목표로 하는가?

원문은 “출력” 또는 “report”만 언급한다.

확인 필요:

- 콘솔/CLI 출력인가?
- 웹 화면인가?
- PDF/CSV 같은 파일인가?
- 인쇄 가능한 문서인가?

상태: `Needs Verification`

## 5. 데이터 갱신 주기와 책임

### Q-005. 분기별 갱신 데이터는 누가 입력하는가?

투자 예상 수익과 운영 비용은 보통 연 4회 갱신된다고 설명한다.

확인 필요:

- 갱신 담당자는 관리자, 투자 고문, 시스템 운영자 중 누구인가?
- 과거 이력 보존이 필요한가, 아니면 최신 값만 저장하는가?

상태: `Needs Verification`

## 6. 금액 정밀도와 반올림

### Q-006. 금액 계산의 반올림 규칙은 무엇인가?

모기지, 세금, 보험, 보조금, 가용 자금 계산은 금액 단위를 다룬다.

확인 필요:

- 센트 단위 반올림 방식
- 주간 환산 시 소수점 처리
- 보고서 표시 방식

상태: `Needs Verification`

## 7. 날짜와 주간 기준

### Q-007. “매주 초”와 “매주 말”의 기준일은 언제인가?

주간 계산과 투자 시점이 요구사항에 등장한다.

확인 필요:

- 주의 시작 요일
- 주의 종료 요일
- 공휴일/영업일 처리 여부

상태: `Needs Verification`

## 8. 기존 모기지의 예상 상환액

### Q-008. 총 예상 모기지 상환액은 모든 활성 모기지의 주간 P&I와 고객 부담금을 포함하는가?

모기지 데이터에는 `Weekly principal and interest payment`, `Current combined gross weekly income`, 세금, 보험료가 있다.

확인 필요:

- 총 예상 모기지 상환액은 재단이 받는 총액인가?
- 부부가 부담하는 금액만인가?
- 보조금 지급 전 전체 모기지 금액인가?

상태: `Needs Verification`

## 9. 시스템 형태

### Q-009. 파일럿 시스템의 기대 실행 형태는 무엇인가?

과제 요구사항에는 구체적인 플랫폼이 없다.

확인 필요:

- CLI 프로그램
- 데스크톱 앱
- 웹 앱
- 단일 사용자 로컬 프로그램

상태: `Needs Verification`

### Q-010. 교수자/과제 평가에서 필수 산출물 형식이 있는가?
