# 01. Problem Definition — The MSG Foundation System

## 1. 배경

Martha Stockton Greengage의 유산을 바탕으로 설립되는 MSG Foundation은 젊은 부부가 주택을 구입할 수 있도록 100% 모기지와 보조금 지원을 제공한다. 재단 이사회는 운영 비용을 줄이고 주간 자금 가용성을 일관되게 계산하기 위해 파일럿 소프트웨어 시스템 도입을 검토하고 있다.

## 2. 해결하려는 문제

MSG Foundation은 매주 다음을 판단해야 한다.

- 투자 수익과 운영 비용을 고려했을 때 이번 주에 주택 구입에 사용할 수 있는 자금이 얼마인가?
- 기존 모기지 상환액과 보조금 지급액을 고려했을 때 신규 모기지를 승인할 수 있는가?
- 투자, 운영 비용, 모기지 데이터를 기반으로 필요한 보고서를 생성할 수 있는가?

현재 문제는 이 계산과 보고가 수작업으로 이루어질 경우 다음 위험이 있다는 점이다.

- 계산 오류 가능성
- 승인 가능 자금 판단의 일관성 부족
- 투자/운영비/모기지 데이터 변경 이력 추적 어려움
- 주간 보고와 요청 보고서 생성에 소요되는 운영 비용 증가

## 3. 목표 상태

파일럿 시스템은 MSG Foundation이 주간 자금 계산에 필요한 최소 데이터만 관리하면서 다음을 수행할 수 있어야 한다.

1. 투자 데이터, 운영 비용 데이터, 모기지 데이터를 기록한다.
2. 매주 사용 가능한 모기지 자금을 계산한다.
3. 기존 모기지의 주간 상환액과 재단 보조금 예상치를 계산한다.
4. 신규 주택 구입 비용이 해당 주 가용 자금 안에 있는지 판단할 수 있는 정보를 제공한다.
5. 주간 자금 결산서, 투자 목록, 모기지 목록 보고서를 생성한다.

## 4. 목표 사용자

- MSG Foundation 관리자
- MSG Foundation 이사회 또는 수탁자
- 투자 고문
- 파일럿 시스템 운영자

## 5. 핵심 비즈니스 가치

- 주간 자금 계산의 정확성 향상
- 모기지 승인 판단의 근거 확보
- 보고서 생성 시간 감소
- 추후 재단 전체 운영 전산화 여부를 판단할 수 있는 파일럿 근거 확보

## 6. 비목표

현재 분석/설계 단계 및 파일럿 범위에서 다음은 비목표로 둔다.

- 실제 금융 거래 실행
- 실제 대출 계약서 생성/전자서명
- 신청자 전체 접수/심사 워크플로우 완전 자동화
- 세금 신고서나 급여 명세서 파일 저장/검증 자동화
- 투자 매매 실행
- 실시간 은행/보험/세무 기관 연동
- 배포, 운영 서버 구축, 실서비스 출시

## 7. 성공 기준

분석/설계 단계의 성공 기준은 다음과 같다.

- PDF 요구사항의 핵심 규칙이 요구사항 ID로 추적된다.
- 주간 자금 계산에 필요한 입력 데이터와 산출 데이터가 명확하다.
- 보고서 3종의 목적과 포함 정보가 정의된다.
- 모호한 산식과 정책은 `Needs Verification`으로 분리된다.
- 구현 단계로 넘길 수 있는 요구사항, 데이터 모델, 인터페이스, 테스트 전략이 준비된다.
